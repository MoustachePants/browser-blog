Stage 1a

The HTML stage

After the browser gets its html file from the network, first, it reads and processes the html in a process called parsing i.e. translates it to a structure that the browser can understand and manipulate. it uses the DOM API to create the finished product of this stage - the DOM tree

[HTML code] ! you can edit the code to see the new DOM tree and also change the finished web page we will render.

[DOM tree]

the dom tree can be manipulated by using the DOM API in js.

! when you are using document.querySelector(“.red-background”).innerHTML = “hello world” you are actually not changing the html source code, but a property of the DOM tree created by the browser

Stage 1b

The CSS stage

! This stage begins after the browser fetches the css file. it can be at the same time as stage 1a

If not getting into sophisticated details, this stage main job is very similar to stage 1a. The browser parses the css file. Now, instead of the DOM API, it uses another API called CSSOM API and creates the CSSOM tree.

[css code editor]

[CSSOM tree]

! it is less common to know the CSSOM API in comparison to the DOM API, but some of us used it without knowing. For example, when we code document.querySelector(“p”).style.backgroundColor = “red” it's the DOM API that calls all of the paragraph elements in the page but the CSSOM API that does all the work changing the background color.
